# 工业齿轮箱3D可视化

这是一个基于Web的工业齿轮箱3D可视化应用，使用Three.js实现3D模型渲染，Tailwind CSS实现响应式UI设计。该应用允许用户360°旋转查看齿轮箱模型，调整模型颜色、缩放比例和光照效果，提供了直观的产品展示和技术交流平台。

**最新更新：应用现已支持用户上传自定义GLTF/GLB格式的3D模型文件，提供更灵活的可视化体验。**

## 功能特点

- **360°旋转查看**：支持全方位旋转查看模型的每个细节
- **真实3D模型加载**：支持加载GLTF格式的真实3D模型文件，提供更真实的可视化体验
- **用户模型上传**：支持用户上传自定义的GLTF/GLB格式3D模型文件
- **拖放上传**：支持通过拖放方式快速上传模型文件
- **动画支持**：如果模型包含动画，可以播放、暂停和切换不同的动画
- **自定义外观**：可根据需求调整模型颜色、材质和光照效果
- **交互控制**：提供直观的控制面板，支持调整旋转速度、缩放比例和光照强度
- **响应式设计**：适配不同屏幕尺寸，提供良好的移动端体验
- **技术信息展示**：展示齿轮箱的基本参数、应用领域、结构特点和性能参数

## 如何使用

### 基本操作
1. 打开 `index.html` 文件即可在浏览器中运行应用
2. 使用鼠标操作模型：
   - 左键拖动：旋转模型
   - 滚轮：缩放模型
   - 右键拖动：平移视图
3. 使用控制面板调整模型参数：
   - 旋转速度：调整模型自动旋转的速度
   - 缩放比例：调整模型的大小
   - 模型颜色：切换模型的颜色
   - 环境光照：调整场景的光照强度
4. 点击"重置视图"按钮恢复模型默认状态

### 模型加载
应用提供两种方式加载3D模型：

#### 1. 默认模型
应用启动时会自动加载一个示例3D模型。

#### 2. 上传自定义模型
应用支持用户上传自己的GLTF/GLB格式3D模型文件：

1. 点击界面右上角的"上传模型"按钮
2. 在弹出的模态框中，可以：
   - 点击"选择文件"按钮，从本地文件系统中选择模型文件
   - 或直接将模型文件拖放到指定区域
3. 确认文件格式正确后，点击"上传"按钮
4. 等待模型加载完成，应用会自动调整相机位置以适应新模型

**支持的文件格式**：
- GLTF (.gltf)
- GLB (.glb)

**注意事项**：
- 大型模型文件可能需要较长的加载时间
- 建议使用优化过的模型，以确保良好的性能
- 如果模型包含纹理，请确保纹理文件已嵌入模型中或与模型文件一起上传

### 动画控制
如果加载的模型包含动画，控制面板会自动显示动画控制选项：
- 播放/暂停按钮：控制动画的播放状态
- 动画选择下拉框：如果模型包含多个动画，可以切换不同的动画

## 技术实现细节

### 模型加载
应用使用Three.js的GLTFLoader来加载GLTF格式的3D模型。GLTF是一种高效的3D模型格式，被称为"3D的JPEG"，具有文件小、加载快的特点，非常适合Web应用。

加载过程包括：
1. 创建GLTFLoader实例
2. 配置DRACOLoader用于处理压缩的模型文件
3. 设置加载进度回调，显示加载进度
4. 加载完成后，处理模型并添加到场景中
5. 如果模型包含动画，创建动画混合器并设置动画控制

### 用户模型上传
应用实现了完整的文件上传功能，包括：

1. 文件选择：通过文件选择对话框或拖放方式选择文件
2. 文件验证：检查文件格式是否为支持的GLTF或GLB格式
3. 文件加载：使用URL.createObjectURL创建临时URL，然后通过GLTFLoader加载模型
4. 资源管理：加载完成后释放临时URL，避免内存泄漏
5. 用户界面：提供直观的上传进度显示和错误处理

### 拖放功能
应用实现了拖放上传功能，主要包括：

1. 阻止默认的拖放行为
2. 添加视觉反馈，提示用户可以放置文件
3. 处理放置的文件，提取文件对象
4. 将文件传递给文件处理函数进行验证和加载
### 错误处理
应用包含完善的错误处理机制：
- 如果模型加载失败，会显示错误提示
- 3秒后自动创建一个简单的占位模型，确保应用可以正常运行
- 所有交互操作都有对象存在性检查，避免空引用错误

### 性能优化
- 使用DRACO压缩算法减小模型文件大小
- 实现模型的自动缩放和相机适配，确保模型正确显示
- 优化渲染循环，只在必要时更新动画和场景

## 注意事项

- 应用默认加载的是示例模型，您可以通过上传功能替换为自己的齿轮箱3D模型
- 应用需要在支持WebGL的现代浏览器中运行（如Chrome、Firefox、Edge等）
- 移动端设备可能需要更强大的硬件支持才能获得流畅的体验
- 加载大型3D模型时，可能需要较长的加载时间，请确保提供良好的加载体验
- 模型文件大小建议控制在合理范围内（推荐不超过50MB），以确保良好的加载速度和性能
- 如果您的模型包含自定义材质或特殊效果，可能需要额外的代码来正确处理这些效果
- 上传的模型文件仅在当前会话中有效，刷新页面后将恢复为默认模型
- 出于安全考虑，浏览器可能会限制某些操作，如访问本地文件系统中的纹理文件
